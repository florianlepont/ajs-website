# Phase 12 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note

This contract covers the two new route trees this phase ships: the Éditions **overview** page (vertical editorial list, `editions/index.astro` + `en/` twin) and the Éditions **detail** page (`editions/[slug].astro` + `en/` twin). It introduces **zero new design tokens** — every color, spacing, and typography value below is an existing site-wide token (`src/layouts/BaseLayout.astro`'s `:root`), read directly from the live codebase this session. Layout decisions D-01 through D-10 are locked in `12-CONTEXT.md` and are not re-litigated here; this document turns them into concrete CSS values, mirroring `02-UI-SPEC.md`'s and the About/Contact pages' established "editorial page" conventions rather than inventing a new visual language. Nav wiring (the "Éditions" main-nav link) is explicitly Phase 13's scope, not this document's.

## Color

**Zero new colors.** Every value is the existing site-wide token set (`BaseLayout.astro` `:root`, White/Ink/Pop-Pink identity, live since Phase 04.1):

| Role | Value | Usage in this phase |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (`--gray-0` / `--color-dominant`) | Overview page background, row background, detail-page content background — unchanged |
| Secondary (30%) | `#FFFFFF` (`--color-secondary`, merged with dominant per D-04's site-wide precedent) | No separate secondary surface is introduced this phase |
| Accent (10%) | `#D6327C` (`--pink-600` / `--color-accent`) | Reserved for: row title link hover/focus-visible underline color (inherited from global `a` rule), focus-visible outline on the hero trigger and back-link, focus-visible outline on grid thumbnail buttons (all pre-existing global rules, not new ones) |
| Destructive | unused this phase | No destructive action exists on these pages (public, read-only showcase) |

**Text/ink color:** `#1A1A1A` (`--color-ink` / `--gray-900`) for all body copy, row titles, format-details text, back-link text — unchanged token, ≈15:1 contrast on white (already verified site-wide).

**Detail-page hero scrim (reused, unchanged token values):** `linear-gradient(to top, rgba(0,0,0,0.65) 0%, rgba(0,0,0,0.15) 45%, rgba(0,0,0,0) 70%)` over the hero photo, white (`#FFFFFF`) title text — identical to `.gallery-detail__hero-scrim`/`.gallery-detail__hero-title`, since the édition detail hero is a structural mirror of the gallery detail hero (D-05/D-06/D-07 only change its *interactivity*, not its visual treatment).

**No commerce-adjacent color use (EDN-06 boundary):** no price/stock/badge color is declared anywhere in this phase — there is no "sold out" red, no price-tag styling, no availability pill. This is a deliberate absence, not an oversight; do not introduce one.

Accent reserved for: link hover/focus states and focus-visible outlines only — never a background fill, never the format-details line's color, never a "new/featured" badge (none exists).

## Copywriting Contract

All copy is bilingual (FR/EN) per EDN-07, following the site's established pattern: FR authored first (primary market), EN as direct professional translation. Every string below is either taken verbatim from CONTEXT.md (D-09's exact example) or derived from the existing gallery-page copy patterns (`02-UI-SPEC.md`'s Copywriting Contract), swapping "gallery" for "édition" vocabulary.

| Element | Copy |
|---------|------|
| Overview page `<h1>` | **FR:** "Éditions" · **EN:** "Editions" |
| Overview eyebrow | **FR/EN:** "Atelier Jacqueline Suzanne" (unchanged across locales, matches About page's identical eyebrow) |
| Primary CTA (each row → detail page) | The entire row is the link; visible text is the édition title (proper noun, shared across locales, e.g. "Rebut") plus the full statement. A screen-reader-only suffix supplies the actionable verb, mirroring the gallery-card pattern exactly: **FR:** "Voir l'édition" · **EN:** "View edition" (concatenated into the link's accessible name, e.g. "Rebut — View edition"). |
| Detail-page hero click (D-05, new interaction — no gallery equivalent) | No visible button copy — the photo itself is the trigger, matching the thumbnail-click convention. Accessible name: **FR:** "Voir en taille réelle, image 1 sur {total}" · **EN:** "View full size, image 1 of {total}" (`{total}` = `images.length + 1`, counting `leadPhoto`) |
| Detail-page grid thumbnail click | Same pattern, offset by the D-06 Lightbox index shift: **FR:** "Voir en taille réelle, image {n} sur {total}" · **EN:** "View full size, image {n} of {total}" (`{n}` = `i + 2`, i.e. the 1-based *displayed* position — `images[0]` is the 2nd photo overall since `leadPhoto` is the 1st) |
| Format details line (D-09, exact strings from CONTEXT.md) | **FR:** "Pages : {pageCount} · Tirage : {printRun} exemplaires · Dimensions : {width} × {height} {unit}" · **EN:** "Pages: {pageCount} · Print run: {printRun} copies · Dimensions: {width} × {height} {unit}" |
| "Retour aux éditions" back-link (D-08) | **FR:** "← Retour aux éditions" · **EN:** "← Back to Editions" (leading arrow glyph, `aria-hidden="true"`, not read twice by screen readers — text alone is the accessible name) |
| Empty state heading (overview, no published éditions — CMS robustness, not expected at launch since Phase 11 seeded real content) | **FR:** "Éditions à venir" · **EN:** "Editions coming soon" |
| Empty state body | **FR:** "De nouvelles éditions seront bientôt visibles ici. Revenez prochainement." · **EN:** "New editions will appear here shortly. Check back soon." |
| Error state | Slug not found → reuse the existing base-aware localized 404 page (`src/pages/404.astro`), unchanged — no new copy needed, same as the gallery detail page's 404 fallback. |
| Destructive confirmation | None. No destructive action exists on these visitor-facing, read-only pages. |
| **EDN-06 negative-copy check (explicit, not implicit)** | No string on either page may contain: a currency symbol (`€`, `$`), or (case-insensitive, FR/EN) "prix", "price", "acheter", "buy", "panier", "cart", "stock", "disponib*", "availab*", "sold out", "épuisé". This applies to every hardcoded string in this contract above — all have been checked against this list. The planner/executor must not introduce any string outside this contract without re-checking it against this list (RESEARCH.md's Pitfall 5 / Validation Architecture already recommends an automated grep guard for this). |

## Layout Notes (Phase 12 specifics, not covered by the template's fixed sections)

### Overview page (`editions/index.astro` + `en/` twin) — vertical editorial list, D-01 through D-04

- **Page frame:** reuses the shared `--editorial-page-*` tokens exactly as About/Contact/legal pages do: `width: min(100%, var(--editorial-page-max))`, `margin: 0 auto`, `padding: var(--editorial-page-padding-block) var(--editorial-page-padding-inline)`.
- **Page header:** eyebrow ("Atelier Jacqueline Suzanne", Label role, uppercase) + `<h1>` ("Éditions"/"Editions", Editorial Page-title role), `border-top: var(--border-hairline) solid var(--color-ink)` above the header block — mirrors `.about-page__hero`'s top-hairline-then-heading structure, minus the portrait/lead-paragraph columns (not requested by CONTEXT.md — a genuinely new, simpler page, not a copy of About's two-column hero).
- **Row structure (D-02):** each row is a two-column grid on desktop (≥768px): `grid-template-columns: minmax(200px, 320px) minmax(0, 1fr)` for a photo-left row, reversed to `minmax(0, 1fr) minmax(200px, 320px)` for a photo-right row (D-04 zigzag — alternate by row index, e.g. `nth-child` even/odd or a `data-flip` boolean computed in frontmatter). Column gap: `var(--space-xl)` (32px).
  - `leadPhoto` thumbnail: fixed `aspect-ratio: 3 / 4` (portrait, book-cover feel — deliberately distinct from the gallery grid's `1:1` square convention, since éditions are printed objects, not framed photography), `object-fit: cover`, rendered via `thumbnailUrl`/`responsiveThumbnailSrcSet` from `src/lib/image.ts` (unchanged helpers).
  - Text column: édition title (Editorial Section-title role) as the row's link text, then the full statement (Body role, `max-width: 640px` within the column, per D-03 no truncation — row height is intentionally variable between éditions).
- **Row divider & rhythm:** `border-top: var(--border-hairline) solid var(--color-ink)` per row (same hairline weight/color as `.about-page__section`'s divider — a deliberately strong, editorial-catalog divider, not a faint `--color-border` hairline), `padding-top: var(--space-lg)` (24px), `var(--space-2xl)` (48px) gap between rows including the divider's own top margin. First row's divider sits `var(--space-2xl)` below the page header.
- **Whole-row is the link** (`<a>` wrapping the row's grid, matching the gallery-card whole-card-link convention) — hover/focus-visible: `outline: 2px solid var(--color-accent); outline-offset: var(--focus-ring-offset);` on the row (existing global focus-visible pattern, no new ring style), plus `transform: scale(1.01)` on the thumbnail only (subtler than the gallery card's `1.02`, since this is a smaller, secondary photo element within a text-forward row, not a full-bleed cover image).
- **Mobile (<768px):** collapses to a single column — photo above text, full-width thumbnail (`aspect-ratio: 3/4` retained, capped `max-width: 320px` to avoid an oversized portrait thumbnail on narrow phones), `gap: var(--space-md)` between the stacked photo and text. **Zigzag does not apply on mobile** — D-04's left/right alternation is a side-by-side-only concept; every row stacks identically (photo-then-text) below the breakpoint, matching `GalleryGrid.astro`'s single existing `768px` breakpoint convention (CONTEXT.md's discretion note).
- **Empty state (0 published éditions):** `<EmptyState heading={...} body={...} />` in place of the row list — reuses the component verbatim, no new styling.

### Detail page (`editions/[slug].astro` + `en/` twin) — hero + statement + format + grid + Lightbox, D-05 through D-10

- **Structure mirrors `galleries/[slug].astro` almost exactly** (same `.gallery-detail`-equivalent wrapper, hero height/scrim/title treatment, `.gallery-detail__content` padding, `GalleryGrid` for the thumbnail grid) — three deltas, all locked by CONTEXT.md:
  1. **Hero is clickable (D-05):** the hero `<img>` is wrapped in a `<button type="button" data-gallery-thumb data-index="0">` instead of being a bare, non-interactive `<img>`. Add `cursor: pointer` and the hover cue `transform: scale(1.02)` on the image only (matches the gallery-card hover precedent, signals interactivity). Add the small expand-affordance SVG icon (see Design System → Icon library) positioned `absolute`, `bottom: var(--space-lg); right: var(--space-lg);`, `opacity: 0.9`, white stroke, inside the hero — a discoverability cue since "click the hero photo" is a new interaction pattern this site has never asked visitors to do before (gallery's hero has always been static).
  2. **Format details line (D-09/D-10):** a single `<p>` between the statement and the thumbnail grid, Label role (14px), `margin: var(--space-2xl) 0` (48px above/below, matching the statement block's existing bottom-margin rhythm), plain text — no table, no icons, no per-field visual separation beyond the `·` (middle-dot) separator character already specified in the exact copy strings above.
  3. **"Retour aux éditions" back-link (D-08):** placed in **normal document flow** as the first child of `.edition-detail__content` — i.e. *below* the full-bleed hero, *above* the statement — never absolutely positioned, never inside the hero/scrim area. This is the deliberate fix for the precedent bug (`10-UAT.md`'s `header-backhome-overlap-logo`): that bug came from an absolutely-positioned link competing for the same screen region as `SiteHeader`'s logo inside the transparent-header hero zone. Placing this link in-flow, entirely outside the hero, makes that class of overlap structurally impossible rather than requiring careful pixel coordination. Styled as a plain `.nav-link`-equivalent (Label role, 14px, `min-height: var(--tap-target-min)` tap target, leading `←` glyph), `margin-bottom: var(--space-lg)` above the statement.
- **Photo-shoot grid (D-07):** `GalleryGrid` wraps `edition.images.map(...)` **with no `.slice(1)`** (unlike gallery's grid, which skips `images[0]`) — every element of `images[]` gets its own thumbnail tile, since `leadPhoto` is a separate field never duplicated into this array. `data-index` on each thumbnail button is `i + 1` (not `i`), per D-06's combined-array offset. Thumbnail visual treatment (1:1 square, `thumbnailUrl`/`responsiveThumbnailSrcSet`, focus-visible outline) is otherwise byte-for-byte identical to `.gallery-detail__thumb`/`.gallery-detail__thumb-button` — no new CSS class needed beyond the `data-index` offset, which is a frontmatter/template concern, not a style concern.
- **Lightbox (D-06):** receives `images={[edition.leadPhoto, ...edition.images]}` — no visual changes to `Lightbox.astro` itself (reused as-is, per EDN-03 and RESEARCH.md Pattern 3).
- **SEO/social image fallback:** since `edition` has no `seo` field/group (confirmed absent in `sanity/schemas/edition.ts`, per RESEARCH.md Pitfall 3), the page `<title>`/meta description/social image are built directly from `edition.title` / `edition.statement[locale]` / `edition.leadPhoto` — this is a data-layer concern, not a visual one, but is flagged here so the executor doesn't reach for a nonexistent `edition.seo?.title`.

### Cross-cutting

- **No new breakpoint.** Both pages use the site's single existing `768px` convention (`GalleryGrid.astro`, `.chrome-band`, `.about-page__hero`) — no new mobile/tablet/desktop tier is introduced.
- **No new interactive component.** The only genuinely new interaction pattern in this phase is the clickable hero (D-05) — everything else (row links, back-link, grid thumbnails, Lightbox) reuses an existing, already-implemented interaction primitive.
