- **Issue:** `npm run build` fails at the Vite/rolldown static-build step with `[rolldown:vite-resolve] ... Caused by: Tsconfig not found astro/tsconfigs/strict`, before any page is even transformed.
- **Root cause investigation:** Confirmed this reproduces byte-identically on the unmodified pre-plan tree (temporarily restored the three deleted files via `git checkout --`, reran `npm run build`, same failure). This is a sandbox/native-binding-level incompatibility in the pinned `astro@7.0.6` + `vite@8.1.3` combination, not a project code or config defect — `package-lock.json` has zero diff from `npm install`, confirming the installed tree matches what's committed and what CI's `npm ci` would install.
