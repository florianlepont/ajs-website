<decisions>
## Implementation Decisions

### Logo hover behavior (item 1)
- **D-01:** The homepage's own logo (`.home-logo` in `HomeCarousel.astro`) currently has no hover/focus state — it's a static swap based on `data-display-mode` only (white logo in carousel mode, black logo in grid mode). It must gain the same interaction pattern `BaseLayout.astro`'s `.logo-mark` already has: a dark chip (`.logo-mark__chip`) that fades in behind the mark on hover/focus-within, cross-fading from the default logo image to the hover-variant image (150ms opacity transition, two stacked `<img>` elements).
- **D-02:** Apply this per display mode the same way `BaseLayout` applies it per header variant: grid mode (static black-on-white bar) behaves like `.site-header--solid` — default black logo, hover/focus crossfades to white-on-dark-chip. Carousel mode (white logo over the hero photo, already has a dark gradient scrim behind it) behaves like `.site-header--transparent` — white logo shown by default, **no hover swap** (matches BaseLayout's existing transparent-header treatment, which intentionally skips the hover effect since the logo already reads clearly over the dark scrim). Do not force an identical hover effect in both modes — mirror BaseLayout's own mode-conditional behavior.

### Remove the /galleries page (items 2 and part of 6)
- **D-03:** Delete `src/pages/galleries/index.astro` and `src/pages/en/galleries/index.astro` entirely. The homepage grid view (`.home-grid` in `HomeCarousel.astro`) is now the sole gallery-browsing entry point — confirmed explicitly: "delete the page entirely," not just unlink it.
- **D-04:** Remove the "Galleries" nav item everywhere it appears:
  - `BaseLayout.astro` line 116 (`<a href={galleriesHref} class="nav-link">{galleriesLabel}</a>`) plus its now-unused `galleriesLabel`/`galleriesHref` consts (lines ~41/43) — this nav renders on About/Contact/gallery-detail/legal pages.
  - `HomeCarousel.astro` line 87 (`<a href={galleriesHref} class="home-nav-link">{galleriesLabel}</a>`) in the homepage's own header nav.
- **D-05:** `HomeCarousel.astro`'s `ctaHref` (currently `= galleriesHref`, used by the "Discover other galleries →" button in the carousel accent panel) can no longer point at a deleted route. Since its purpose was "see the other galleries," and that's now the grid view on the same page, repurpose the CTA to switch the homepage into grid display mode (i.e., trigger the same client-side `showGrid()` behavior as the mode-toggle button) rather than navigating to a URL.
- **D-06 (gallery-detail "back" links):** `src/pages/galleries/[slug].astro` (line 35/54) and `src/pages/en/galleries/[slug].astro` (line 32/51) each have a `backHref = getRelativeLocaleUrl(locale, 'galleries')` powering a "← Retour aux galeries" / "← Back to galleries" link in the gallery-detail hero. Since `/galleries` no longer exists, point `backHref` at the homepage (`getRelativeLocaleUrl(locale, '')`) instead, and update the visible label text to match (e.g., "← Retour à l'accueil" / "← Back home") — don't leave stale copy that says "galleries" while linking home.
- **Gallery detail pages themselves (`[slug].astro`) are NOT deleted** — only the `/galleries` *listing* page goes away. Detail pages remain the link target for both carousel and grid mode (see D-09/D-10 below).

### Mode toggle: icons instead of text (item 3)
- **D-07:** Replace the "Carousel" / "Grid" text buttons (`toggleCarouselLabel`/`toggleGridLabel`, `.home-toggle__btn`) with icon-based buttons. Same `data-action="show-carousel"`/`data-action="show-grid"` behavior and `is-active` state styling — only the button content changes from text to an icon. Icon choice/implementation (inline SVG vs. icon font) is Claude's Discretion (see below). **Locked constraint:** the button's accessible name must stay exactly `Carrousel`/`Grille` (FR) and `Carousel`/`Grid` (EN) — i.e. add `aria-label={toggleCarouselLabel}` / `aria-label={toggleGridLabel}` (or equivalent visually-hidden text) rather than dropping the label. `tests/e2e/homepage.spec.ts` (lines 36, 57, 68) locates these buttons via `page.getByRole('button', { name: 'Grille' })` / `{ name: 'Carrousel' }` — those tests must keep passing unmodified; this is a visual change only, not an accessible-name change.

### Mobile hero visibility bug (item 4)
- **D-08:** Real-device report: the homepage hero visually disappears on phone viewports. The existing `@media (max-width: 767px)` rule already sets `.home-hero { height: auto; min-height: 600px; overflow: visible; }` (added during the prior `homepage-visual-bugs` debug session for a *different*, already-resolved issue — see Code Context below) — that fix is not sufficient on its own for this report. Treat this as a real bug requiring investigation (e.g., check for a parent/ancestor with `overflow: hidden` or a zero-height flex/grid context clipping the hero, a stacking-context issue, `100vh` mobile-viewport quirks bleeding through from the base carousel style, or JS `render()` not being called before layout), not just another guessed CSS value. Verify the fix against a real mobile viewport (devtools device emulation at minimum; real device if available), not just by reading the CSS.

### Three-line wordmark (item 5)
- **D-09:** The wordmark constant `wordmark = 'Atelier Jacqueline Suzanne'` (single string, single line today) must render as three explicit lines — "Atelier" / "Jacqueline" / "Suzanne" — in both places it appears: `.home-hero__wordmark` (carousel accent panel) and `.home-grid__wordmark` (grid intro band). Use explicit line breaks (not text-wrapping) since the split must always be exactly these three words on three lines regardless of container width.

### Clickable gallery names (item 6)
- **D-10 (carousel mode):** The hero caption title (`data-role="gallery-title"`, currently a plain `<h2>`) must become a link to that specific gallery's detail page (`getRelativeLocaleUrl(locale, `galleries/${slug}`)`, same pattern grid tiles already use). Note the carousel's client-side `render()` function re-renders this element's `textContent` on every auto-advance/prev/next — the fix must keep the href in sync with the currently-displayed gallery's slug, not just make the initial server-rendered title clickable. The build-time data island (`<ul data-role="home-carousel-data">`) currently only carries `title`/`heroSrc`/`alt` per gallery — it will need a `slug` added so the client script can compute the right href on each render.
- **D-11 (grid mode):** `.home-grid__tile` is already a full `<a href={.../galleries/${gallery.slug}}>` wrapping the image, scrim, and title — confirmed already fully clickable end-to-end. No change needed here; the plan should include a verification step (not a code change) confirming this still holds after the other changes land.

### Claude's Discretion
- Exact icon glyphs/style for the carousel/grid toggle (inline SVG is consistent with the rest of the codebase's dependency-free approach — no icon library is currently installed).
- Root-cause technique for the mobile hero bug investigation.
- Whether the repurposed "Discover other galleries →" CTA (D-05) keeps its arrow/copy as-is or is reworded slightly now that it triggers a mode switch instead of navigation — keep it close to the existing copy unless it reads wrong once wired up.

</decisions>

<code_context>
## Existing Code Insights

### Reusable assets
- `BaseLayout.astro`'s `.logo-mark` / `.logo-mark__chip` / `.logo-mark__img--default` / `.logo-mark__img--hover` CSS pattern (two stacked `<img>`, opacity crossfade, dark chip) — reuse this exact mechanism for `.home-logo` rather than inventing a new hover pattern.
- `.home-grid__tile`'s existing `<a>`-wrapping-everything pattern is the template for making the carousel title clickable (D-10) — same href helper (`getRelativeLocaleUrl(locale, 'galleries/' + slug)`), just applied to the hero caption instead of a grid tile.

### Established patterns
- Client-side rendering in `HomeCarousel.astro` uses a hidden `<ul data-role="home-carousel-data">` data island read once by the `<script>` block, then an in-memory `galleries` array + `render()` function mutates the DOM on prev/next/auto-advance (no framework, no `client:*` directive — see the component's own header comment for why). Any per-gallery field the client script needs (like `slug` for D-10) must be added to this data island, mirroring how `title`/`heroSrc`/`alt` are already threaded through.
- Base-aware asset paths (`import.meta.env.BASE_URL`) are used for the logo `<img>` srcs already in both `BaseLayout.astro` and `HomeCarousel.astro` — preserve this pattern for any new/changed asset references.

### Integration points
- `HomeCarousel.astro` is invoked by `src/pages/index.astro` and `src/pages/en/index.astro` with a `galleries` prop already fetched/filtered at build time via `getGalleries()` — no Sanity client access happens in the component itself (threat-model note already in the file's header comment); this phase should preserve that boundary (don't add a `src/lib/sanity` import to the component or its client `<script>`).
- **`src/components/GalleryCard.astro` becomes dead code once D-03 lands** — it is imported ONLY by `src/pages/galleries/index.astro` and `src/pages/en/galleries/index.astro` (both being deleted). Delete it too rather than leaving an unused component (project convention — see CLAUDE.md: "If you are certain that something is unused, you can delete it completely"). Its `' — Voir la galerie' / ' — View gallery'` accessible-name suffix (line 24) does not need to be preserved anywhere else.
- **`src/components/GalleryGrid.astro` must NOT be deleted** — despite the similar name, it's a generic content-agnostic layout wrapper reused by BOTH the listing page (being deleted) AND the gallery-detail page's own thumbnail grid (still needed, unrelated to this phase).
- **`tests/e2e/gallery.spec.ts` (all 3 test blocks) currently bootstrap via `page.goto('/galleries/')` then `page.getByRole('link', { name: /voir la galerie/i })`** to discover a real slug/href before exercising the detail page and lightbox. Once D-03 deletes that route AND `GalleryCard.astro` (whose copy produces the "voir la galerie" accessible name), this discovery step breaks. Rewrite it to start from `/` (or `/en/`), switch to grid mode if needed, and locate a gallery link via `.home-grid__tile` (or an equivalent accessible query against its actual accessible name — the tile is an `<a>` wrapping an `<img alt=...>` + a `<span class="home-grid__tile-title">`, not GalleryCard's "— Voir la galerie" suffix, so the matcher text must change, not just the URL).
- `tests/e2e/homepage.spec.ts` already asserts against the toggle buttons' text-based accessible names (see D-07 lock) and against carousel/grid structural roles — read it in full before touching `HomeCarousel.astro` so none of its existing assertions break incidentally.
