## Phase Goal

**As a** first-time visitor, **I want to** see the homepage chrome appear instantly and each photo resolve smoothly from a blurred placeholder to sharp, **so that** I never face a blocking full-screen loading state while the images load.

*(User story derived faithfully from the existing ROADMAP Phase 9 `**Goal:**` line — role, capability, and outcome are all present in that goal's own wording; nothing invented.)*

<objective>
Deliver HOME-09 as two vertical slices behind a single failing-test task: (1) the hero photo loads with priority and blur-to-sharp on every swap, with the next photo prefetched; (2) grid tiles get the same blur-to-sharp treatment while staying lazy. The page shell already renders synchronously with no loader — this phase must simply never introduce one (a negative/regression contract).

Purpose: First-time visitors see a uniformly polished, progressive image experience across the whole homepage instead of photos popping in abruptly, without any blocking full-screen loading state.

Output: `blurPlaceholderUrl()` helper, a `blurSrc` field threaded build-time → client, two-layer opacity-crossfade markup/CSS/JS for hero and grid, `fetchpriority="high"` on the hero, next-photo prefetch, and a new `progressive image loading (HOME-09)` e2e block that all of the above turns green.
</objective>
