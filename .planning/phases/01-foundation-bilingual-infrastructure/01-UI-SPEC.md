# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Phase Scope (from ROADMAP.md / CONTEXT.md)

This is a **walking-skeleton** phase — the only UI surfaces are:

1. **Site-wide chrome** (header nav + footer) rendered by `BaseLayout.astro`, sourced at build time from the Sanity `siteSettings` singleton (D-09) — nav labels, footer text, and site title are CMS-driven content, not hardcoded strings, but the *typography/color/spacing* that renders them is locked by this contract.
2. **Language switcher** — plain-text "FR | EN" toggle in the header (D-11, locked — no dropdown, no flags).
3. **Placeholder homepage** — bare, locale-aware short welcome text (D-10) at `/` (French) and `/en/` (English). No real portfolio/about/contact content yet.
4. **404 page** — a real prerendered `404.astro` wired via `public/.htaccess` (`ErrorDocument 404`), bilingual, since Apache serves nothing useful by default (RESEARCH.md "Don't Hand-Roll").

No galleries, product cards, forms, or lists exist yet — later phases (2-5) extend this same chrome and typography/color contract rather than replacing it. Treat every value below as durable baseline, not throwaway.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — shadcn deliberately not initialized this phase |
| Preset | not applicable |
| Component library | none (plain semantic HTML + hand-written CSS; no Radix/base-ui) |
| Icon library | none (text-only chrome per D-11 — no flag icons, no nav icons) |
| Font | System font stack (placeholder): `-apple-system, BlinkMacSystemFont, "Segoe UI", Helvetica, Arial, sans-serif` |

**Why no shadcn:** RESEARCH.md (Standard Stack / Supporting, Assumption A3) explicitly recommends deferring any component-framework install (`@astrojs/react`/`@astrojs/preact`) — Phase 1's entire interactive surface is two `<a>` tags with build-time-computed hrefs plus a ~10-line vanilla cookie script (Pattern 3). shadcn's CLI targets a React/Vite component workflow that doesn't exist in this phase's architecture. Revisit shadcn when Phase 2 (galleries/lightbox) needs a real interactive component library — do not retrofit it into Phase 1's static chrome.

**Why a placeholder font:** No brand/art-direction decision has been made yet (CONTEXT.md has no font/visual-identity decision — this phase is infrastructure-only). Using the OS default system-font stack avoids a premature, uninformed brand commitment and avoids adding a web-font dependency/license cost before Romane's portfolio identity work happens in Phase 2+. Flag for revisit: once real visual identity is established, this token should be replaced project-wide (single point of change: `BaseLayout.astro`'s root CSS).

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/inline gaps (e.g. gap between "FR" and "\|" and "EN") |
| sm | 8px | Compact element spacing (switcher link padding, footer line spacing) |
| md | 16px | Default element spacing (nav item gaps, paragraph spacing) |
| lg | 24px | Section padding (header/footer vertical padding on mobile) |
| xl | 32px | Layout gaps (header/footer vertical padding on desktop, homepage content block spacing) |
| 2xl | 48px | Major section breaks (space above/below homepage placeholder text block) |
| 3xl | 64px | Page-level spacing (not heavily used this phase — reserved for later content-heavy pages) |

Exceptions: Language switcher tap target must be a minimum **44px** height (WCAG 2.5.5 / mobile touch-target minimum) even though the visible text is small — apply via padding, not font-size inflation. 44px is a multiple of 4, so it fits the scale as a floor constraint rather than a new token.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 14px | 400 | 1.5 |
| Body | 16px | 400 | 1.5 |
| Heading | 20px | 600 | 1.2 |
| Display | 32px | 600 | 1.2 |

- **Label (14/400/1.5):** footer copy, legal/copyright line, switcher link text ("FR", "EN", "\|" separator).
- **Body (16/400/1.5):** homepage placeholder welcome paragraph, 404 page body copy.
- **Heading (20/600/1.2):** site title in header (from `siteSettings.title`), 404 page heading.
- **Display (32/600/1.2):** homepage placeholder welcome heading (D-10).

Exactly 4 sizes, exactly 2 weights (400 regular, 600 semibold) — no other size/weight combination is permitted this phase. Later phases add roles (e.g. gallery captions) by reusing these same 4 sizes/2 weights, not introducing new ones, unless a future UI-SPEC explicitly revises this table.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Main page background — homepage content area, 404 page background |
| Secondary (30%) | `#F5F4F2` | Header band background, footer band background (visually separates chrome from content) |
| Accent (10%) | `#1A1A1A` | Reserved for: link text color (switcher "FR"/"EN", any nav link), link hover/active underline, current-locale emphasis in the switcher, `:focus-visible` outline color |
| Destructive | `#DC2626` | Not used in this phase's UI (no destructive actions exist yet) — forward-declared now so Phase 3's contact-form error states and any future delete/confirm actions inherit a single consistent value rather than each phase picking its own red |

Accent reserved for: switcher link text/hover/underline, the currently-active locale's visual emphasis (e.g. bold or underline on "FR" when on a French page), and focus-visible outlines on all interactive elements. Never apply accent to body copy, backgrounds, or decorative elements.

This is a deliberately monochrome (grayscale) palette for Phase 1 — no brand accent color has been chosen yet (no CONTEXT.md decision on this). Photography-portfolio sites conventionally keep chrome neutral so imagery (arriving in Phase 2) provides the color. Revisit if/when a brand color is chosen for the portfolio identity.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Language switcher | "FR \| EN" (locked, D-11 — plain text, no dropdown, no flags; each half is a link, not a toggle button) |
| Primary CTA | None exists in Phase 1. The homepage is placeholder welcome text only (D-10) with no linked action — the first real CTA ("Voir les galeries" / "View galleries") arrives in Phase 2 when portfolio content exists. Do not add a placeholder CTA button this phase. |
| Homepage welcome (FR) | Short, locale-aware placeholder welcome text sourced from Sanity — not hardcoded. Example tone: "Bienvenue sur le site de l'Atelier Jacqueline Suzanne. Le site est en cours de construction." (exact final copy is a CMS content decision for Romane, not fixed here — this contract governs *how* it renders, not the literal words) |
| Homepage welcome (EN) | English counterpart of the above, same CMS document, same rendering rules. |
| Empty state | Not applicable this phase — no lists, galleries, or data-driven collections exist yet to be empty. First real empty-state need arrives in Phase 2 (e.g. a gallery with zero images) or Phase 3 (contact form submission states). |
| Error state (404 Not Found) | FR: "Page introuvable. Retournez à l'accueil." / EN: "Page not found. Return home." — rendered via `404.astro` at Heading (20/600) + Body (16/400), with a link back to that locale's homepage styled per the Accent color rule above. |
| Destructive confirmation | Not applicable — no destructive actions exist in Phase 1 (no delete/remove UI; CMS auth/editing happens inside Sanity Studio, outside this project's rendered UI). |

## Layout Notes (supports Visuals dimension)

- **Header:** single row, site title (Heading role) on the leading side, language switcher (Label role) on the trailing side. Secondary-color band background, `lg`/`xl` vertical padding, full-width.
- **Footer:** Secondary-color band background, Label-role text (copyright/site name from `siteSettings`), `lg` vertical padding, centered or leading-aligned text, no columns/links grid needed yet (nothing to link to beyond home).
- **Homepage placeholder:** Dominant-color background, single centered or leading-aligned text block (Display heading + Body paragraph), generous vertical whitespace (`2xl` above/below) since there is no other content to balance against.
- **404 page:** Same BaseLayout chrome (header + footer persist), Dominant background, centered text block (Heading + Body + one Accent-colored link home).
- Both locale variants (`/` and `/en/`) render this exact same layout — only the copy differs, never the structure, so I18N-01's "correctly localized UI chrome" success criterion is a content swap, not a layout fork.
