## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser (untrusted client) ↔ static site | Client executes the inline cookie/redirect script and the switcher click handler |
| Sanity content → rendered HTML | CMS-authored strings are baked into static HTML at build time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-COOKIE | Information Disclosure (privacy) | `ajs_locale` locale-preference cookie | mitigate | Cookie stores only `fr`/`en`, no visitor identifier; `SameSite=Lax`, 1-year max-age (RESEARCH.md Security Domain). Flagged to Phase 4 for CNIL cookie-consent disclosure assessment per D-03 — do NOT resolve consent here. |
| T-01-XSS | Tampering | siteSettings strings rendered into chrome/homepage | mitigate | Render all CMS strings via Astro's default auto-escaping; `set:html` is forbidden on siteSettings fields (asserted in acceptance criteria). |
| T-01-REDIR | Tampering | inline locale redirect script | mitigate | Redirect target is a same-origin path computed by prefixing `/en` to the current pathname — no user-controlled or external URL is ever used as the redirect destination (no open-redirect surface). |
</threat_model>
