# Phase 04.1 Plan 01: BaseLayout Rebrand Foundation Summary

**Repainted BaseLayout.astro's :root tokens to the white/ink/pop-pink identity with self-hosted Archivo Black, and restructured the header/footer to white-on-white chrome with an AJS logo hover-invert and a headerVariant prop.**

## Performance

- **Duration:** ~35 min of execution (plus a blocking human-verify checkpoint pause for package-legitimacy sign-off before Task 2's `npm install`)
- **Completed:** 2026-07-09
- **Tasks:** 3 (1 checkpoint + 2 auto tasks)

## Accomplishments
- Installed `@fontsource/archivo-black` (self-hosted display font, no Google Fonts CDN request) after a blocking package-legitimacy checkpoint verified the `fontsource` org, `github.com/fontsource/font-files` repo, v5.2.8, and 50,411 weekly downloads
- Swapped BaseLayout's entire `:root` token block to the white/ink/pop-pink identity (base values, decorative palette, semantic aliases, typography roles, radii/border tokens), removed `--color-destructive` entirely, and deleted the dormant Delight `@font-face` + its `deferred-items.md` tracking entry

- Header/footer merged onto white-on-white chrome, delineated only by a 1px `--color-border` hairline; replaced the text site-title with the AJS wordmark PNG logo, hover-inverting to the white variant over a dark backdrop chip
- Added a `headerVariant?: 'solid' | 'transparent' | 'none'` prop to `BaseLayout`'s `Props` interface (default `'solid'`) so downstream Wave-2 plans (homepage, gallery-detail) can opt into the transparent hero-scrim header or suppress the header entirely

**2. [Rule 2 - Missing Critical] Base-aware logo asset paths**
- **Found during:** Task 3 (logo `<img>` markup)
- **Issue:** The plan's example markup used bare `/logos/...` src paths, which would 404 on GitHub Pages' non-root base (`/ajs-website/...`) — this exact class of bug was previously found and fixed for the 404 page in Phase 1's post-completion review (CR-01).
- **Fix:** Added an inline `assetBase` helper in `BaseLayout.astro`'s frontmatter (`(import.meta.env.BASE_URL ?? '/').replace(/\/$/, '')`) and prefixed both logo `src` values with it.
- **Files modified:** `src/layouts/BaseLayout.astro`
- **Verification:** `npm run build` output paths correct under the local root base; matches the existing `stripBasePath`/`i18n-paths.ts` convention used elsewhere in the codebase.
- **Committed in:** `5b62668` (Task 3 commit)

**3. [Rule 1 - Bug] Literal "Delight"/"color-destructive" strings survived in explanatory code comments**

- **Issue:** The new `:root` block's comments referenced the removed identity/token by name for context, which technically kept the literal strings in the file even though the actual `@font-face` and CSS custom property were gone.
- **Fix:** Reworded both comments to describe what was removed without repeating the literal token/font names, satisfying the plan's explicit `grep -c ... returns 0` acceptance criteria while keeping the rationale documented.
- **Files modified:** `src/layouts/BaseLayout.astro`
