# Phase 04.1 Plan 04: Homepage Carousel/Grid Summary

**The homepage is now the real gallery entry point — a full-viewport hero carousel with rotating decorative accent panel, a carousel/grid toggle, and auto-advance/pause — replacing Phase 1's "under construction" placeholder, and the Wave-0 RED spec is GREEN.**

## Performance

- **Duration:** ~50 min (execution was interrupted mid-Task-2 by an account API spend limit, then resumed and completed by the orchestrator directly)
- **Completed:** 2026-07-09

## Accomplishments
- Built `HomeCarousel.astro`, a dependency-free vanilla-JS island implementing carousel mode (full-bleed hero + rotating accent panel + prev/next + index label), grid mode (2-column seamless tile grid), a toggle between them, and 6s auto-advance that pauses on hover/focus — no framework, mirroring the existing `Lightbox.astro`/`LanguageSwitcher.astro` island pattern.
- Replaced `index.astro`/`en/index.astro` with the real homepage: `getGalleries()` filtered generically by `images.length > 0` (D-12), rendered inside `BaseLayout headerVariant="none"`.
- Turned the Wave-0 RED spec (`tests/e2e/homepage.spec.ts`, Plan 04.1-02) fully GREEN, and verified `i18n.spec.ts` still passes (homepage `<header>` still carries the FR|EN switcher).
- Discovered and fixed a genuine content-drift finding: a third Sanity gallery document (slug `adults`) now has real published photos under the title "Paysage" — added directly in Sanity Studio, outside this phase's work. The D-12 filter correctly and automatically picked it up (proving the "generic, not a hardcoded denylist" design intent works as designed).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug/gap in existing code/tests] Two locator bugs in the Wave-0 RED spec surfaced while turning it GREEN**
- **Found during:** Task 2 verification
- **Issue:** (a) The D-12 "all migrated galleries reachable" test read `body.innerText()` in default carousel view, but a carousel shows only the current slide's title by design — the assertion could never pass regardless of implementation correctness. (b) The toggle test's `page.getByText(/silos/i).first()` matched DOM order, not visibility, and resolved to the now-hidden carousel `<h2>` (which also says "SILOS") instead of the visible grid tile.
- **Fix:** Scoped both checks to the grid container (`page.locator('[data-role="home-grid"]')`) — grid mode renders every gallery as its own simultaneously-visible tile, which is what "reachable"/"visible" should actually mean for a carousel UI.
- **Files modified:** `tests/e2e/homepage.spec.ts`
- **Verification:** `npx playwright test tests/e2e/homepage.spec.ts tests/e2e/i18n.spec.ts` — 12/12 pass
- **Committed in:** `bb04e9a`

**2. [Rule 1 — Bug/gap] Live content drift: a third real gallery now exists**
- **Found during:** Task 2 verification (index label showed "01 / 03", not "01 / 02")

- **Fix:** Updated the lists to `[Silos, Brume, Paysage]` migrated / `[Rebut, The Victorian Tea Room]` still-unmigrated, with a code comment explaining the content-driven (not hardcoded-slug) filter is working as designed.

## Issues Encountered

**Worth flagging (not a defect):** the homepage no longer offers a direct path to About or Contact from its own header — visitors reach them via Galeries → any gallery-detail or listing page → standard header. This matches 04.1-UI-SPEC.md's Layout Notes precisely (the imported prototype's intentional minimal/immersive homepage nav), so no code change was made, but it's a real one-extra-click UX change worth Romane/the developer being aware of if user feedback ever flags it.
