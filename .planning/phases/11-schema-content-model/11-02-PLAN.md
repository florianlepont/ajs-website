## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-02 | Elevation of Privilege | Sanity write credentials during seeding | medium | mitigate | Recommended path seeds through the interactive hosted Studio session (the editor's own authenticated session, no standalone token). If the scripted `createOrReplace` fallback is used instead, follow the Phase 1 precedent exactly: `sanity tokens add --role=editor` → use once → `sanity tokens delete`, and verify via `sanity tokens list` that only the permanent viewer token remains — no long-lived write token left behind. |
| T-11-01 | Tampering | Incomplete `dimensions`/`leadPhoto` published to production | medium | mitigate | Validation spot-check in Task 2 confirms Plan 01's parent-level `dimensions` custom validator and `leadPhoto` `assetRequired()` actually block incomplete publishes before the data reaches Phase 12's build-time fetch. |
| T-11-SC | Tampering | npm installs | low | accept | No packages installed this phase; deploy uses the already-installed Sanity CLI. No new supply-chain surface. |
