## Nav Integration Contract (phase-specific)

This section is the actual visual/interaction contract this phase needs — more concrete than the generic template sections below, which mostly confirm "no change."

### Nav order (D-01, locked)

```
Éditions → À propos → Contact → Instagram (icon)
```

"Éditions" renders as the **first** `<a class="nav-link">` inside `<nav class="site-nav">`, before the existing `aboutHref` link — not appended at the end, not inserted between existing links.

### New props (both call sites: `BaseLayout.astro` line ~186, `HomeCarousel.astro` line ~114)

`SiteHeader.astro`'s `Props` interface gains:

```ts
editionsLabel: string;
editionsHref: string;
```

Rendered as:

```astro
<nav class="site-nav" aria-label="Primary">
  <a href={editionsHref} class="nav-link">{editionsLabel}</a>
  <a href={aboutHref} class="nav-link">{aboutLabel}</a>
  <a href={contactHref} class="nav-link">{contactLabel}</a>
  {/* Instagram <a> unchanged */}
</nav>
```

No new CSS class is needed — `.nav-link` (existing) applies unchanged: `display: inline-flex; align-items: center; min-height: var(--tap-target-min); padding: var(--space-xs) var(--space-sm); font-size: 14px; font-weight: var(--weight-regular); line-height: 1.5; text-decoration: none;` plus the existing `:hover`/`:focus-visible` underline. Do not create an `.nav-link--editions` variant class — this would be a design-system violation (asymmetric styling with no visual justification; the CONTEXT.md decisions only ask for order + copy differences, never a different visual treatment).

### Copy source (mirrors `aboutLabel`/`contactLabel` exactly)

- `src/lib/site-config.ts` `resolveSiteCopy()` gains: `editionsLabel: settings?.navLabels?.editions?.[locale] || 'Éditions'` (same literal fallback string in both `fr` and `en` — see Copywriting Contract below).
- `src/layouts/BaseLayout.astro` and `src/components/HomeCarousel.astro` each compute `const editionsHref = getRelativeLocaleUrl(locale, 'editions');` next to their existing `aboutHref`/`contactHref` consts, and pass both `editionsLabel`/`editionsHref` into `<SiteHeader>`.
- `sanity/schemas/siteSettings.ts` `navLabels` object gains an `editions` field, copy-pasted from the existing `about`/`contact` field shape (lines 139–157): `{name: 'editions', title: 'Lien Éditions', type: 'object', fields: [{name:'fr', title:'Français', type:'string'}, {name:'en', title:'Anglais', type:'string'}]}`. Add `editions: {fr: 'Éditions', en: 'Éditions'}` to the schema's `initialValue.navLabels` alongside the existing `about`/`contact` defaults, matching the established pattern exactly.

### Active-state treatment: none (Claude's Discretion, confirmed)

No nav link anywhere on the site currently gets a "current page" visual treatment (confirmed by reading `SiteHeader.astro` in full — no `aria-current`, no active class). "Éditions" gets the same treatment: **no active-state styling**, for consistency with À propos/Contact. Do not introduce this as a novel pattern scoped only to Éditions.

### Mobile fit (D-02/D-03, locked)

The header now carries a 4th nav link. Existing breakpoints in `SiteHeader.astro`'s `<style is:global>` block:

- `@media (max-width: 767px)`: `flex-wrap: wrap; gap: var(--space-xs);` on `.site-header`/`.site-nav`, tightened link padding. **Re-measure live** after adding the 4th link — do not assume the existing trims are still sufficient; this is a live-measured methodology (Phase 7/10 precedent), not a formula.
- `@media (max-width: 359px)`: `flex-wrap: nowrap; gap: 0;` with `font-size: 12px` and 2px horizontal padding on `.nav-link`. This is the **last-resort tier**. Exhaust these existing trims first before reaching for label abbreviation.
- **New, scoped exception (D-03):** if the existing sub-359px squeeze still doesn't fit all 4 links + logo + language switcher on one row, abbreviate "Éditions" to a short recognizable form (e.g. `Éd.` — CONTEXT.md's working example; executor may pick any short recognizable form). This abbreviation applies **only** to the Éditions link at this one breakpoint — À propos/Contact/Instagram are never abbreviated, at any breakpoint, anywhere on the site. Verify the chosen abbreviation's legibility live at actual device size (same "verify live" precedent as the Instagram icon and Phase 10 mobile work) before locking it in.
- Every nav link — including the abbreviated form, if used — retains the `--tap-target-min` (44px) minimum vertical tap target. Abbreviation trims character count, never touches tap-target height.
- No two-line wrap on the header at any width (D-02 explicitly rejects this) — the header stays a single row from the widest viewport down to the narrowest supported phone.

### Explicit non-goals (do not implement in this phase)

- Do not add "Éditions" to `HomeCarousel.astro`'s carousel rotation or grid tiles — the homepage's photography content stays pure photography (Phase 11 D-13, restated as this phase's success criterion #3). This phase only touches the `<SiteHeader>` nav, never `getEditions()`/`getEdition()` data wiring into the carousel/grid.
- Do not touch the Éditions overview/detail page templates (Phase 12, already shipped) or the `edition` Sanity schema (Phase 11, already shipped).
- Do not add any pricing/availability/purchase affordance anywhere this phase touches (not applicable to a nav link, but stated for completeness per the milestone's no-commerce boundary).

## Color

Declared roles — **already established, unchanged by this phase**. The "Éditions" link introduces no new color.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-dominant` = `--gray-0` = `#FFFFFF` | Page/header background (solid variant); merged with secondary per D-04 |
| Secondary (30%) | merged into Dominant (`--color-secondary: var(--gray-0)`) | Header/footer chrome — deliberately white-on-white, no separate secondary surface |
| Accent (10%) | `--color-accent` = `--pink-600` = `#D6327C` | Body copy links, focus rings, per-gallery hero accent panels — **NOT** used for nav-link text/hover state |
| Destructive | none defined; not applicable to this phase (no destructive action exists in nav wiring) | — |

**Nav link color contract:** `.nav-link` uses `color: inherit` from its parent `.site-header--solid` (ink, `#1A1A1A`) or `.site-header--transparent` (white, `#FFFFFF`) — it never uses the accent pink. The new "Éditions" link follows this identical rule; do not give it a distinct/accent color to visually signal its new "priority" nav position — D-01's priority is expressed through **position** (first in the nav), not through color, matching the project's existing "header links are neutral chrome, not accent actions" convention (see `SiteHeader.astro` comment above `.nav-link` color rules).

Accent reserved for: body-copy inline links, `:focus-visible` outline ring (`--color-focus-ring`), per-gallery hero color panels on the homepage carousel/grid (`HERO_COLORS` in `site-config.ts`). Never the header nav links, including the new Éditions link.
