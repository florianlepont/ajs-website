<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| visitor's browser → Web3Forms API | Untrusted form input (name/email/message) leaves the client via `fetch()`; there is no server-side compute on this project, so all gating happens client-side or at the SaaS. |
| bot/automated agent → the form | Automated spam submissions are the primary abuse vector against the free-tier quota and Romane's inbox. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Denial of Service | ContactForm submit handler (bot spam) | mitigate | Client-side honeypot short-circuit (CONT-02, D-12): if `isHoneypotTriggered(website)` the handler returns before `fetch()`, discarding spam silently. Verified by the e2e honeypot-no-request test + the honeypot human-check. Locked scope — no CAPTCHA/rate-limiting added. |
| T-03-02-02 | Spoofing / Denial of Service | Direct POST to api.web3forms.com with the public access key (bypassing the site) | accept | Un-preventable on a static host with no server-side rate limiting (03-RESEARCH.md Security Domain). Web3Forms' own free-tier quota + bot detection (verified to reject non-browser requests) is the only backstop; acceptable for a low-traffic single-artist site. |
| T-03-02-03 | Information Disclosure | PUBLIC_WEB3FORMS_ACCESS_KEY inlined into client JS | accept | Public/embeddable identifier by design (Web3Forms' own docs embed it in plain HTML); it is not a secret and grants no account access. PUBLIC_ prefix is intentional. No mitigation required. |
| T-03-02-04 | Tampering (input validation, ASVS V5) | name/email/message fields | mitigate | Client-side required-field + email-format checks (`isBlank`, `isValidEmail`) before the network call. Message content is only ever relayed to email, never rendered back into any page in this codebase, so stored/reflected XSS is not a realistic vector this phase (flag for any future "view submissions" feature). |
| T-03-02-05 | Tampering | src/layouts/BaseLayout.astro (shared chrome) | mitigate | Additive nav link only; full e2e suite gate confirms switcher/cookie/header contracts unbroken. |
| T-03-02-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages this phase (03-RESEARCH.md: Web3Forms is a `fetch()` call, no SDK). Package Legitimacy Gate not applicable. |
</threat_model>
