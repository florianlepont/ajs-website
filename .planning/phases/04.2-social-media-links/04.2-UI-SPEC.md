## Copywriting Contract

All copy is bilingual (FR/EN) per I18N-01. The Instagram handle itself (`@ajs_romanelepont`) is not
translated in either locale ‚Äî it is a proper-noun handle. Target URL (confirmed convention):
`https://www.instagram.com/ajs_romanelepont/`.

| Element | Copy |
|---------|------|
| Footer link (site-wide, both locales) | Visible text: **"Instagram"** (unlocalized brand name, same FR/EN ‚Äî no translation needed, matches how brand names are conventionally left as-is). Rendered with `.nav-link`, styled identically to the existing footer legal-nav links. Includes a visually-hidden new-tab hint using the already-shipped `.sr-only` utility class: **FR:** "(nouvelle fen√™tre)" ¬∑ **EN:** "(opens in new tab)" ‚Äî appended inside the `<a>` after the visible text, so the accessible name reads "Instagram (opens in new tab)" / "Instagram (nouvelle fen√™tre)". |
| About page sentence (closing line, after the medium/technique paragraph) | **FR:** "Suivez le travail de Romane sur Instagram : @ajs_romanelepont" (the handle is the link) ¬∑ **EN:** "Follow Romane's work on Instagram: @ajs_romanelepont" ‚Äî same `.sr-only` new-tab hint appended to the link as the footer instance. |
| Contact page sentence (placed after the lead paragraph, before the `<ContactForm>`) | **FR:** "Vous pouvez aussi me suivre sur Instagram : @ajs_romanelepont" (the handle is the link) ¬∑ **EN:** "You can also follow me on Instagram: @ajs_romanelepont" ‚Äî same `.sr-only` new-tab hint appended. |
| Primary CTA | Not applicable in the classic "action button" sense ‚Äî this phase's only interactive element is the Instagram link itself (see rows above); there is no form submission or button introduced. |
| Empty state | Not applicable ‚Äî no data-driven/list view exists in this phase. |
| Error state | Not applicable ‚Äî this is a static outbound link with no app-controlled failure mode (an unreachable instagram.com is outside this site's control and not a state this UI needs to handle). |

## Layout Notes (phase specifics, not covered by the template's fixed sections)

### Footer (site-wide ‚Äî `BaseLayout.astro`)
- The footer currently renders `footer-text` + one `<nav class="footer-legal-nav" aria-label="Legal">`. This phase adds a **second, separate** `<nav class="footer-social-nav" aria-label="Social">`, placed **after** `footer-legal-nav` in DOM order (not merged into the legal nav ‚Äî semantically distinct, and appending rather than reordering avoids touching the DOM position of the already-tested legal links).
- `aria-label="Social"` is hardcoded (unlocalized), matching the existing unlocalized `aria-label="Legal"` convention already shipped on the sibling nav ‚Äî this is consistent with current codebase practice, not a new inconsistency introduced by this phase.
- Styling: reuse `.nav-link` for the `<a>` itself; the new `<nav>` gets the same `margin-top: var(--space-md); display: flex; gap: var(--space-xl);` rule as `.footer-legal-nav` (copy the existing rule under a new class name, or generalize both under one shared class ‚Äî implementation detail left to the executor, visual result must be identical spacing).
- Because `BaseLayout.astro`'s `<footer>` renders unconditionally regardless of `headerVariant` (including on the homepage, whose header uses `headerVariant="none"`), adding the link here satisfies the "site-wide" half of the ROADMAP goal with a single-file change ‚Äî no per-page wiring needed.

### About page (`about.astro` / `en/about.astro`)
- Insertion point: a new closing `<p>` after the existing medium/technique placeholder paragraph (the last element in `.about-page`), using the same `.about-page p` styling already applied to every other paragraph on the page (no new class).
- This is a **content** addition, not a structural one ‚Äî no new heading, no new section wrapper.

### Contact page (`contact.astro` / `en/contact.astro`)
- Insertion point: a new `<p>` between the existing `<p class="contact-page__lead">` and the `<ContactForm locale={locale} />` mount ‚Äî visible above the fold, before the visitor reaches the form, framing Instagram as an alternative contact channel.
- Do not reuse the `contact-page__lead` class name for this new paragraph (that class is semantically "the form's intro sentence"); give it its own minimal rule that matches the same Body-role font-size/weight/line-height (16px/400/1.5) with its own `margin: 0 0 var(--space-2xl);` (matching the lead's existing bottom margin so the form's vertical rhythm is unchanged).

### Bilingual pairing
- Every visible string above has a locked FR/EN pair. The handle `@ajs_romanelepont` and the brand word "Instagram" are intentionally identical across locales (proper nouns), consistent with how `siteTitle` ("Atelier Jacqueline Suzanne") is already left untranslated site-wide (`BaseLayout.astro` comment, Phase 1 Plan 04).
