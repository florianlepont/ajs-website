### `src/layouts/BaseLayout.astro` — footer (component, request-response)

**Analog:** the existing `.footer-legal-nav` `<nav>`, same file.

**Current footer markup** (`src/layouts/BaseLayout.astro:122-128`):
```astro
<footer class="chrome-band">
  <p class="footer-text">{footerText}</p>
  <nav class="footer-legal-nav" aria-label="Legal">
    <a href={legalNoticeHref} class="nav-link">{legalNoticeLabel}</a>
    <a href={privacyHref} class="nav-link">{privacyLabel}</a>
  </nav>
</footer>
```

**Insertion (per UI-SPEC "Footer" Layout Notes):** append a **second, separate** `<nav>` after
`.footer-legal-nav`, not merged into it:
```astro
<footer class="chrome-band">
  <p class="footer-text">{footerText}</p>
  <nav class="footer-legal-nav" aria-label="Legal">
    <a href={legalNoticeHref} class="nav-link">{legalNoticeLabel}</a>
    <a href={privacyHref} class="nav-link">{privacyLabel}</a>
  </nav>
  <nav class="footer-social-nav" aria-label="Social">
    <a
      href="https://www.instagram.com/ajs_romanelepont/"
      target="_blank"
      rel="noopener noreferrer"
      class="nav-link"
    >
      Instagram<span class="sr-only">{instagramNewTabHint}</span>
    </a>
  </nav>
</footer>
```

**Frontmatter variable pattern to copy** (matches the existing hardcoded locale-conditional
convention used for `aboutLabel`/`contactLabel`/`legalNoticeLabel` — About/Contact/legal copy is
"not CMS-editable," same rationale applies to this brand-name link's `.sr-only` hint):
`src/layouts/BaseLayout.astro:57-60`
```astro
const legalNoticeLabel = locale === 'en' ? 'Legal notice' : 'Mentions légales';
const legalNoticeHref = getRelativeLocaleUrl(locale, 'mentions-legales');
const privacyLabel = locale === 'en' ? 'Privacy' : 'Confidentialité';
const privacyHref = getRelativeLocaleUrl(locale, 'confidentialite');
```
Add alongside these:
```astro
const instagramNewTabHint = locale === 'en' ? ' (opens in new tab)' : ' (nouvelle fenêtre)';
```
Note the href for Instagram is an absolute external URL — it does NOT go through
`getRelativeLocaleUrl` (that helper is only for internal/site-relative routes; every other href in
this file is internal).

**CSS pattern to copy** (`src/layouts/BaseLayout.astro:386-390`):
```css
.footer-legal-nav {
  display: flex;
  gap: var(--space-xl);
  margin-top: var(--space-md);
}
```
UI-SPEC directs: copy this rule under a new class name (`.footer-social-nav`) with identical
`display: flex; gap: var(--space-xl); margin-top: var(--space-md);` — or generalize both nav
classes under one shared rule. Either is acceptable; visual output must be identical spacing to
the legal nav. `.nav-link` itself (`src/layouts/BaseLayout.astro:362-372`) needs zero changes —
it's reused as-is:
```css
.nav-link {
  font-size: 14px; /* Label role */
  font-weight: var(--weight-regular);
  line-height: 1.5;
  text-decoration: none;
}

.nav-link:hover,
.nav-link:focus-visible {
  text-decoration: underline;
}
```

## Shared Patterns

### External link mechanics (href / target / rel)
**Source:** new to the codebase — no existing external (non-`getRelativeLocaleUrl`) link exists yet
in `BaseLayout.astro`, `about.astro`, or `contact.astro`. UI-SPEC locks the exact attributes to use,
identically across all three insertion points:
```astro
href="https://www.instagram.com/ajs_romanelepont/"
target="_blank"
rel="noopener noreferrer"
```
**Apply to:** all 3 Instagram link instances (footer, About, Contact) in both locales (5 total
`<a>` tags across the 5 files).

### `.sr-only` new-tab hint (visually-hidden accessible-name suffix)
**Source:** `src/components/GalleryCard.astro:40` (existing idiom — a `<span class="sr-only">`
placed as the last child inside/adjacent to an `<a>`, to extend the accessible name without adding
visible text) + the utility class definition itself, `src/layouts/BaseLayout.astro:245-255`:
```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```
**Apply to:** all 3 Instagram link instances — `<span class="sr-only">{hint}</span>` (footer, where
the hint is a frontmatter variable per the hardcoded-locale-conditional convention) or a literal
`<span class="sr-only"> (nouvelle fenêtre)</span>` / `<span class="sr-only"> (opens in new tab)</span>`
inline (About/Contact page prose, where copy is already hardcoded per-locale in the page body, not
lifted into a variable — matches how the rest of each page's copy is written directly in the
template, not as frontmatter consts).

### Global link color/hover/focus (no override needed)
**Source:** `src/layouts/BaseLayout.astro:221-241` (global `a` rules, `is:global` style block):
```css
a {
  color: var(--color-accent);
  text-decoration: none;
}

a:hover,
a:focus-visible {
  text-decoration: underline;
  text-decoration-thickness: 2px;
  text-underline-offset: 3px;
}

a:focus-visible,
button:focus-visible {
  outline: 2px solid var(--color-accent);
  outline-offset: var(--focus-ring-offset);
}
```
**Apply to:** all 3 Instagram link instances automatically (they are plain `<a>` tags, no override
class) — UI-SPEC confirms zero new CSS is needed for color/hover/focus anywhere this phase touches.

### Bilingual hardcoded-copy convention (no Sanity/CMS involvement)
**Source:** `src/layouts/BaseLayout.astro:44-51` (About/Contact nav labels — "not in CMS scope...
hardcoded locale conditional") and the FR/EN page-pair pattern itself (`about.astro` /
`en/about.astro`, `contact.astro` / `en/contact.astro` — same structure, only copy and import depth
differ, per each file's own header comment).
**Apply to:** every string this phase introduces (footer link text/hint, About sentence, Contact
sentence) — all hardcoded per-locale, none read from Sanity, consistent with existing precedent that
this content is out of CMS-01's galleries-only scope.
