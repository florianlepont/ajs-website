  <action>
    Create src/components/Lightbox.astro accepting `interface Props { images: GalleryImage[]; locale: 'fr' | 'en' }`. In the template render a single `<dialog id="lightbox" aria-label="...">` containing: a close button (`data-action="close"`, top-right), a prev button (`data-action="prev"`, vertically centered left), a next button (`data-action="next"`, vertically centered right), a centered `<img data-role="lightbox-image">`, and a counter `<p data-role="counter" aria-live="polite">`. Use hand-authored inline SVG icons (24×24 viewBox, `stroke="currentColor"`) for chevron-left, chevron-right, and close/X. Pre-render the full-size sources: emit the `fullSizeUrl(...)` for each image plus its localized alt into a data structure the script reads (e.g. a JSON `<script type="application/json">` block or `data-*` attributes on hidden list items) so the island has the URLs/alt without importing the build-time image module at runtime.
    In a plain `<script>` (no `client:*`): get the dialog via `getElementById`; maintain a `current` index; a `render()` that sets the image `src`/`alt` and the counter text ("{n} / {total}"); `showPrev()`/`showNext()` that wrap or clamp within the gallery; an `open(index)` that sets `current`, calls `render()`, then `dialog.showModal()` (this is what gives native focus containment + Escape-to-close — do NOT hand-roll a focus trap or an Escape listener, Anti-Pattern). Wire click handlers on prev/next/close (close via `dialog.close()`). Add a `keydown` listener on the dialog for `ArrowLeft`→showPrev / `ArrowRight`→showNext (arrow-key image nav is domain-specific and must be hand-written per Pitfall 1). Add touchstart/touchend swipe detection (50px threshold, require horizontal dominance `Math.abs(dx) > Math.abs(dy)`), `{passive: true}`. Preload the neighbouring image on navigation for perceived performance. On the dialog's `close` event, return focus to the thumbnail that opened it (store the trigger element in `open()`); the trigger wiring itself is added in Task 2.
    Expose the open function to the page: attach it to thumbnails found via the `data-gallery-thumb` hook — but since the thumbnails live in the page (Plan 03), the cleanest contract is for this script to `querySelectorAll('[data-gallery-thumb]')` and bind each thumbnail's click to `open(Number(el.dataset.index))`, storing `el` as the return-focus target. Document this hook contract in a comment.
    Scoped `<style>`: scrim `background: rgba(20, 18, 19, 0.96)` (Woodsmoke, NOT the pink accent), full-viewport; image `max-width`/`max-height` viewport-constrained with `object-fit: contain` (never cropped); all three controls `min-width: 44px; min-height: 44px` (tap-target, via padding around the 24px icon), icon `color: var(--color-dominant)`; counter at Label role (14px/300). Use tokens, not raw hex, except the scrim rgba which is a documented exception.
    Do NOT build custom broken-image error UI — native broken-image + required alt text suffices (UI-SPEC YAGNI note).
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Static HTML + client island → visitor browser | The lightbox runs client-side over already-rendered static markup; no server round-trip, no network input. |
| Sanity → build | Image URLs/alt embedded at build time (unchanged read boundary). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-CLIENT | Tampering | Lightbox client `<script>` | accept | Pure DOM interaction over build-time-embedded data; no user input, no eval, no network calls, no secrets. Alt/URL strings are Astro-escaped at build. |
| T-02-04-A11Y | Denial of Service (usability) | Native `<dialog>` focus/Escape | mitigate | Use `showModal()` for native focus containment + Escape; e2e asserts focus returns to trigger on close (avoids the hand-rolled-focus-trap bug class). |
| T-02-04-IV | Tampering (Input) | Gallery image/alt content | accept | Same Studio-side validation as 02-01/02-03; no new input surface. |
</threat_model>
