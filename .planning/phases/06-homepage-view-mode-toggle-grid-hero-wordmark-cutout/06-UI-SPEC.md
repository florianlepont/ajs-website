## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |

Introducing shadcn now, mid-project, on a single vanilla-CSS component would be inconsistent with six already-shipped phases — skipped per the gate's own scoping (tech stack is not React/Next.js/Vite), not asked.

Existing site-wide tokens (`src/layouts/BaseLayout.astro` `:root`) — reuse as-is, no new tokens needed for this phase:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps (current accent-panel/grid-intro padding) |
| 2xl | 48px | Major section breaks (current grid-intro band padding) |
| 3xl | 64px | Page-level spacing |

Exceptions:
- `--tap-target-min: 44px` — already an established token, reused unchanged for the single toggle button's hit area (D-01). Not a new exception.
- The new grid hero-tile (D-04) should use **`--space-md` (16px)** padding, not the old `.home-grid__intro`'s `--space-2xl` (48px) — the tile is now a constrained `aspect-ratio: 1/1` square (same size as every gallery tile), not a full-width band, so it needs tighter padding to leave room for the wordmark + intro copy. Flag as an intentional per-tile exception, not a token change.

## Layout & Interaction Notes

*(Project convention — code comments in `HomeCarousel.astro` already reference "UI-SPEC Layout Notes"; this section is the canonical source for that reference in this phase.)*

### Single toggle button (D-01, D-02, D-03, HOME-01)
- Replace both `.home-toggle__btn` elements with **one** `<button data-role="mode-toggle" data-action="toggle-mode">`, containing **both** existing inline SVG icons as children (reuse the carousel-icon and grid-icon markup verbatim — do not redesign glyphs).
- Icon visibility is driven by `data-display-mode` exactly like `.home-hero`/`.home-grid`'s existing `hidden` toggling: the icon shown is the icon of the mode you'd switch **to** (grid icon visible while in carousel mode; carousel icon visible while in grid mode) — mirrors the label semantics above.
- `aria-label` is set/updated in the same `render()`/mode-switch script path that already flips `data-display-mode`, using the two existing localized strings (`toggleCarouselLabel`/`toggleGridLabel`).
- The wrapping `<div class="home-toggle" role="group" aria-label="Display mode">` should drop `role="group"`/`aria-label` — a `role="group"` implies multiple related controls; with one button it's simpler and more correct to leave it a plain layout wrapper (or remove the wrapping div and place the button directly in `.home-header`, if layout allows).
- `setActiveToggle()`'s `is-active`-class bookkeeping (currently juggling two buttons + excluding the CTA) is no longer needed — delete it. A single button has no "which one is active" state to track.

### Grid hero-tile (D-04, D-05, D-06, HOME-02)
- Delete `.home-grid__intro` as a full-width band. Its content (wordmark + intro paragraph) becomes the first child of `.home-grid__tiles`, in a **non-link** `<div class="home-grid__tile home-grid__tile--hero">` (D-06 — explicitly not an `<a>`; it has no gallery to link to, and must not be implemented as a dead/empty-href link).
- `.home-grid__tile--hero` reuses `.home-grid__tile`'s exact sizing rules (`position: relative; aspect-ratio: 1/1; overflow: hidden`) so it sits identically to every other tile in the 2-column (desktop) / 1-column (mobile) grid — not spanning multiple columns, not otherwise emphasized, per D-04's "simplest, most literal reading."
- Background: solid, cycling accent color (`background-color: var(--current-accent, var(--color-accent))` — reuse the same accent the carousel is currently showing, via the existing `--current-accent` CSS custom property already set on `.home` by `render()`) with `color: var(--color-on-accent)`. No photo, no cutout — D-05 confirms grid mode stays solid-panel treatment.
- Content: `.home-grid__wordmark` (three `<br>`-separated lines, same "Atelier / Jacqueline / Suzanne" text) + `.home-grid__intro-body` paragraph, both resized per the Typography table above to fit the square tile. No CTA, no gallery title, no image.

### Carousel wordmark cutout (D-07, D-08, D-09, HOME-03)
- Scope: `.home-hero__wordmark` only, inside `.home-hero__accent`, carousel mode only. Grid mode is explicitly out of scope for the cutout (D-05/D-07).
- Technique: `background-clip: text` (+ `-webkit-background-clip: text` for Safari), `color: transparent` (+ `-webkit-text-fill-color: transparent`), with `background-image` set to the **currently-displayed gallery's own hero photo** — same `gallery.heroSrc` already assigned to `heroImg.src` in `render()`. Add a matching assignment (e.g. a CSS custom property `--wordmark-photo` set on the wordmark element or `.home-hero__accent`, updated alongside `heroImg.src` in the same `render()` call) so the two always stay in sync per gallery/auto-advance tick.
- Positioning: `background-size: cover; background-position: center;` — the CONTEXT explicitly permits this **simpler full-bleed approximation** over pixel-matching the exact visible crop behind the accent panel (D-08: "Exact CSS approach... is Claude's Discretion"). Do not spend implementation time computing exact offset alignment; verify the simpler approach reads correctly instead.
- `.home-hero__accent`'s own solid `background-color: var(--color-accent)` (currently applied via inline style to the whole panel) must no longer sit behind the wordmark, or the cutout will be invisible. Remove the panel-wide background; instead give **only** `.home-hero__intro` its own backing (a padded box using the same cycling `--current-accent` color the panel used to have, so the intro paragraph keeps its existing contrast/legibility) — per D-08, "the panel's other content — intro paragraph — can keep its own backing/contrast treatment; only the wordmark needs the photo to show through."
- Fallback: wrap the cutout rules in `@supports (background-clip: text) or (-webkit-background-clip: text)`; outside that support (effectively unreachable in 2026 evergreen browsers, but keep the safety net), fall back to solid `color: var(--color-on-accent)` text with no background-image, so the wordmark never renders invisible/blank.
- **Must be verified live** (screenshot/browser check, D-08's explicit requirement): confirm the letters visibly reveal photo detail — not just that the DOM/CSS is "correct" — at both a typical desktop width and a mobile width (375px), for at least two different galleries' photos (since the effect depends on photo content, a photo that happens to be uniformly dark/light near the accent-panel corner could still read as illegible even with correct CSS).

### CTA removal ripple (D-10)
- Delete `.home-hero__cta` CSS block, the `ctaLabel` variable/computation, and the `<button class="home-hero__cta" data-action="show-grid">` markup. Confirm via `grep` after the change that no `ctaLabel`/`.home-hero__cta` references remain.
- Re-check `.home-hero__accent`'s padding/vertical rhythm once the CTA line is gone and the wordmark grows larger (Claude's Discretion per CONTEXT.md) — likely reduce `margin-bottom` on `.home-hero__intro` slightly now that nothing follows it, but this is a minor balance pass, not a hard requirement.

### Test impact (D-02)
- `tests/e2e/homepage.spec.ts` ~lines 36/57/68: update the `getByRole('button', ...)` queries to target the single `[data-role="mode-toggle"]` button — the label **strings** themselves (`'Grille'`/`'Carrousel'`) do not need to change per the Copywriting Contract above, only the fact that there's one button whose accessible name changes with state, not two independently-named buttons. Flag this explicitly as an intentional, expected update in the plan (per D-02), not a regression against Phase 04.3's locked-name decision.
- Any assertion relying on `.home-toggle__btn.is-active` / two-button counts must be removed or rewritten — that DOM structure no longer exists.
