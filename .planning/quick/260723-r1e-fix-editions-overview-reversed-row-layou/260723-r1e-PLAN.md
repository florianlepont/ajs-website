<objective>
Fix the Éditions overview reversed-row layout bug on both locale twins. On alternating (odd-indexed) éditions the photo and text stack vertically instead of sitting side-by-side, because the two grid children have only an explicit `grid-column` (no `grid-row`); the CSS Grid sparse auto-placement algorithm pushes the reversed row's text into a new implicit row 2 when its column index is less than the auto-placement cursor. Root cause is already diagnosed live via DOM inspection — this plan only applies the fix and locks it with a regression test.

Purpose: The zigzag list is visually broken on every odd édition (reported directly by the site owner). This is a pure CSS fix, no markup/JS change.
Output: `grid-row` declarations added to both overview pages, plus a permanent Playwright regression guard in the existing edition spec.
</objective>

Project constraint (CLAUDE.md): the site is bilingual FR/EN — the two overview files carry byte-identical `<style>` blocks for this component and MUST be kept in parity.

Confirmed root cause (do NOT re-diagnose): `.editions-list__row` is a CSS grid. Its children `.editions-list__photo` and `.editions-list__text` set only `grid-column`. Markup order is always photo-then-text. For reversed rows (`.editions-list__row--reverse`, odd index) photo=column 2 and text=column 1; because text's column (1) is behind the auto-placement cursor after photo was placed, the sparse algorithm pushes text into a NEW implicit row 2 instead of sharing row 1. Confirmed: reversed row's `grid-template-rows` computed to two tracks (`426.664px 146.5px`) vs the correct single track (`426.664px`).

The fix: add `grid-row: 1` to both children so auto-placement can never split them across two implicit rows. It applies identically regardless of reverse state (both variants are meant to be a single row).

CRITICAL companion detail — the mobile media query: in `@media (max-width: 767px)` the row becomes `grid-template-columns: 1fr` and BOTH children are forced to `grid-column: 1` (intentional vertical stack). A `grid-row: 1` on the base rules leaks into this single-column layout and would place photo AND text in the SAME cell (row 1 / col 1) → they overlap. The mobile media query must therefore reset `grid-row: auto` for both children so they continue to auto-place into separate stacked rows. This is still only a `grid-row` change — no `grid-template-columns` or reverse-swap logic is touched.
</context>
