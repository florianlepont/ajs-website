<objective>
Make the homepage's grid-mode hero tile text legible against dark per-gallery accent colors (Violet/Plum), completing the per-gallery accent-color pipeline so it is correct in BOTH carousel and grid display modes.

Purpose: A gallery with a dark accent preset currently renders the grid-mode hero tile with dark ink text on a dark background — unreadable. The carousel already handles this correctly; the grid tile is the one surface still hardcoding a light-only text color.

## Diagnosis reconciliation (READ FIRST — the task's root-cause writeup is partly stale)

Direct investigation of the CURRENT code (not the task's writeup) found the pipeline has already been largely fixed since the task was drafted. Confirmed facts:

- `src/lib/site-config.ts` already exists and is the frontend palette module: `HERO_COLORS` (slug→on-brand hex, identical hexes to HeroColorInput.tsx), `normalizeHeroColor(slug)→hex`, and `getHeroTextColor(hex)→'#1A1A1A'|'#FFFFFF'` (WCAG-contrast). Introduced commit `5957583`, 2026-07-13.
- `src/pages/index.astro` (line 38/50-51) and `src/pages/en/index.astro` (line 35/44-45) ALREADY resolve `heroColor` slug → on-brand HEX via `normalizeHeroColor`, and ALREADY compute `heroTextColor` via `getHeroTextColor` BEFORE passing both into `<HomeCarousel>`.
- Therefore `HomeCarousel`'s `gallery.heroColor` is an on-brand HEX (e.g. `#AF3DFF`), NOT the raw slug — so `--current-accent` (line 101) is already correct/on-brand. Bug (1) "off-brand color" is already fixed. The stale screenshot showing CSS-native `purple` (#800080) predates commit `5957583`.
- `gallery.heroTextColor` is already the correct per-gallery contrast color and IS consumed correctly by the carousel accent panel at SSR (line 198) and in `render()` (line 392). It is NOT dead code.

Consequences for the prescribed mechanism in the task writeup:
- DO NOT create `src/lib/hero-colors.ts`. It would be a redundant THIRD copy of the palette, AND it would BREAK the pipeline: it was specced to key by slug, but `HomeCarousel` receives hexes — every lookup would miss and silently fall through to the ACCENTS fallback, regressing the currently-correct carousel.
- DO NOT remove the `heroTextColor` prop/field trail. It is live and correct, and this fix reuses it for the grid tile.

The ONLY genuine remaining defect: the grid-mode hero tile CSS (`.home-grid__tile.home-grid__tile--hero`, line ~1312) hardcodes `color: var(--color-on-accent)` (dark ink) while its background already tracks the live `--current-accent`. On a dark accent the two collide → illegible. This plan fixes exactly that, mirroring the existing `--current-accent` mechanism.
