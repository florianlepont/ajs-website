<objective>
Fix two accessibility findings from the UI audit, both on the homepage stack:

1. **Link-color contrast (site-wide):** `--pink-600: #FF3B94;` in `src/layouts/BaseLayout.astro` (line 222) is the brand pink that flows into `--color-accent` (line 234) and from there into `--color-link`, `--color-link-hover`, and `--color-focus-ring` (lines 238-240) — the text color of every link on the site. Its contrast against the white page background is ~3.33:1, below WCAG AA's 4.5:1 minimum for normal text. Replace the single token value with the pre-verified darker on-brand shade `#D6327C` (~4.56:1 against white), which propagates to every link/hover/focus-ring consumer through the existing `var()` chain.

2. **Homepage semantic heading:** the homepage currently has zero heading elements — the "Atelier Jacqueline Suzanne" wordmark is marked up as a `<p>` in both display modes of `src/components/HomeCarousel.astro` (line 205 carousel, line 216 grid). Promote both wordmark elements from `<p>` to `<h1>`, keeping their classes and text content identical (tag-name-only change). Because the grid container carries `hidden` by default, only one wordmark is ever in the accessibility tree, so this yields exactly one accessible top-level heading at a time.

Purpose: Clear two blocking WCAG AA failures identified in the UI audit so the homepage is accessible and the site's link text is legible.

Output: one-line CSS token change in BaseLayout.astro, two tag swaps in HomeCarousel.astro, plus a new e2e assertion locking in the homepage top-level-heading contract.
</objective>
